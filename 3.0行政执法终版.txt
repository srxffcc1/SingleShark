框架设计-------------------------------------------------------------------------
a1.执法编号页面 添加功能-直接添加方案 条项内容-编号名称+详情按钮+所属计划+所检察企业+可选阶段（可以模糊理解为目前阶段）点击可选阶段跳到a2 点击详情跳到a3
a2.展示当前阶段内容简要内容+下方对阶段进行点亮 下方阶段可以点击 点击则跳转xb 
a3.展示已经存在的阶段 对阶段进行点亮（点亮表示存在阶段数据） 阶段有下拉菜单 和 添加菜单 （已经存在数据则不提供添加 无数据则不提供下拉）底部菜单有修改 删除 隐藏 （修改会另起一个dialog ）删除会进行确认

b基本是dialog
1b.检查计划-制定检查时间-类型-名称 保存是提示添加检查方案
2b.检查方案-填写检查方案 选择检查企业 检查类型 框好检查项 
3b.现场检查-基本信息-检查项viewpager 检查项来源-默认加入来自方案 保存弹dialog 把不合格的筛选出来 右边增加下拉 现场 责令 处罚 并处
4b.现场措施-回退功能 单条回退 下一步进入整改  
5b.责令-回退 单条回退 下一步整改 
6b.整改复查 展示tab来自 现场处理措施 责令 
7b.处罚 来自检查 来自整改复查 
8b.归档 不放在这个位置 

字段设计-------------------------------------------------------------------------
计划{
计划期限
计划类型
计划负责人
计划成员
计划名称
计划涉及企业
}
方案{
被检查单位-选择的
地址-选择带出来的
联系人-选择带出来的
检查时间-选择的
执法人员-选择的
检查内容-为检查项的综合-选取的
检查方式-选择的或者手填
方案需要提请上级进行审核 上级能看到简单页面 批复审核意见 再到下一步审批意见 两位领导吧
至于是不是要方案审核完了才能下一步 可以暂缓
}
现场检查记录{
被检查单位（由方案带入的）
地址-（选择之后带出来的）
法定代表人-（选择之后带出来的）
法定代表人职务和电话-（带出来的）
检查场所-应该是多个部位-（从检查项进行综合提炼）
检查时间
检查人员-进行选择
检查人员证件号码-（选择带出）
安监局所属-（也是选择之后自动判断）
检查情况-检查项提炼
签名-用签名板进行收集-提示为请特定的检查人员进行签名来进行对应
现场负责人签名-直接采集现场负责人签名即可
（转阶段时会对检查记录进行选择要转的阶段）
}
检查项{
合格不合格
默认法律
部位
相关证据
}
现场处理措施{
被检查单位-现场检查带过来的
事故隐患-选择的时候转的 显示一个列表分已编辑 来选择法律依据和相关处理决定
不服决定的部分应该是各个地方定死的
签名还是手写板采集一下或者直接使用历史签名
人员还是选择加签名
}
整改复查{
被检查单位-从现场检查带
复查意见-自己填写
关联的那份文书-给个选择就行 
}
责令整改{
被检查单位-带过来的
存在问题-带过来的 每项问题设置整改期限
不服决定的部分应该是各个地方定死的
人员还是选择加签名
}
处罚决定{
违法列表-证据将会附在后面 自行选择违反了什么法律（应该是自动关联的法律）依据什么什么给予进行收集 （给予处罚应该是罚金）
罚款缴纳地点-可能是某某银行地址或者其他
账号-那就是银行账号了
不服决定的部分应该是各个地方定死的
}
归档{---应该是和执法检查平行目录
即为一个执法检查编号的整体流程查看各个流程的信息和文件只提供查看 可看为记录
}
